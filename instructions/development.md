# Процесс разработки

[Final task](https://github.com/rolling-scopes-school/tasks/tree/master/stage2/modules/final-task)

- **Ветвь разработки - это основная ветвь разработки**, содержащая последнюю рабочую версию проекта, в которую интегрируются все завершенные функциональные компоненты по мере их доработки.

- Каждый член команды, включая руководителя группы, ведет разработку в своей собственной ветке.

- Названия веток соответствуют конкретной реализуемой функциональности. Ветки создаются в начале разработки каждой функции на основе текущего состояния ветки разработки.

- Все слияния в ветку разработки происходят только с помощью Pull-Request. Для разрешения конфликтов слияния требуется объединить ветвь разработки с вашей ветвью компонентов.

- Ветка считается готовой к объединению в develop, если есть хотя бы одно одобрение от другого члена команды или наставника и все комментарии (если таковые имеются) помечены как разрешенные. Комментарий может быть разрешен только тем пользователем, который его оставил.

- После одобрения слияния каждый член команды отвечает за объединение своих собственных PR. При необходимости они разрешают конфликты.

- Ожидается, что всем членам команды будет предложено рассмотреть и утвердить запросы на обновление.

- **Каждый член команды должен внести как минимум пять исправлений и как минимум два Pull Request**, объединенных в ветку разработки. **Несоблюдение этих требований приводит к начислению штрафных баллов за недостатки в организации команды.**
